What I like most about Python, and made me fall desperately in love with it, is how it focuses on writing clean and readable code. We all know how hard it can be to read other people’s code, especially if it is badly formatted. Python tries to avoid that using rules and tips such as the functional imposition of nesting through indentation. This feature alone makes the code much more visual. While in many languages indentation is just a suggestion, in Python becomes a rule; either you indent or it does not work. If there is something that exemplifies perfectly what I’m just saying is the Tim Peters’ Zen of Python,, which is an easter egg that can be found in Python version 2.1.2 or later by typing import this in a Python interpreter

The Zen of Python, by Tim Peters

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
Special cases aren't special enough to break the rules.
Although practicality beats purity.
Errors should never pass silently.
Unless explicitly silenced.
If the implementation is hard to explain, it's a bad idea.
If the implementation is easy to explain, it may be a good idea.
Namespaces are one honking great idea -- let's do more of those!
and few more....

This tells a lot about the Python philosophy, and why I really enjoy writing code in Python. In fact, this principles can be applied to any programming language. Writing good code, clear, readable, understandable depends more on you than on the programming language you are working with.

Steve Jobs stated once that “simple can be harder than complex”. And that's true. Requires a lot of hard work to get your thinking clean, to come up with a simple solution. But a simple solution is always better than a complex one.

Simple communicates better. Simple creates a instant connection. Simplicity is something you can apply in everything in your life – not only programming.

Stay humble. Be a simple person.

- Sagar Maharjan
